## 快速开始（3 步上手）

### 第 1 步：安装依赖

首次使用需要安装以下 Python 依赖：

```bash
pip install pdfplumber pytesseract Pillow pandas openpyxl rich
```

> macOS 额外需要：`brew install tesseract`
> Windows 额外需要：[下载 Tesseract OCR 安装包](https://github.com/UB-Mannheim/tesseract/wiki)

### 第 2 步：初始化设置

```bash
cd scripts/invoice-trip-organizer
python3 setup.py init
```

按提示选择存放目录，脚本会自动创建：

```
个人行程与报销/
├── 01 发票整理/
│   ├── 01 待分类/    ← 放入新发票
│   ├── 02 待核实/    ← 无法识别的文件
│   └── 03 已完成/    ← 已归档发票
├── 02 行程与个人报销单/
│   └── 2026 年/
│       ├── 1 月/
│       ├── 2 月/
│       └── ...
├── scripts/invoice-trip-organizer/  ← 脚本和配置（按 Skill 名称存放）
│   └── invoice-trip-organizer/
├── 总台账.md
└── SOP-发票文件命名标准.md
```

### 第 3 步：开始使用

**发票整理**：将发票放入 `01 待分类/`，然后运行：

```bash
python3 invoice_auto_organizer.py
```

**行程管理**：创建行程并关联发票：

```bash
python3 trip_auto_organizer.py
```

**邮件下载发票**：从邮箱下载发票附件：

```bash
python3 download_invoices.py
```

**上传文件**：通过系统弹窗选择文件：

```bash
python3 upload_files.py
```

## 配置文件说明

初始化后会生成 `config.py`，包含以下配置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `OBSIDIAN_VAULT` | 项目根目录 | 初始化时选择 |
| `EMAIL_ADDRESS` | 163 邮箱地址 | 需手动填写 |
| `EMAIL_AUTH_CODE` | 邮箱授权码 | 需手动填写 |
| `QQ_EMAIL_ADDRESS` | QQ 邮箱地址 | 可选 |
| `QQ_EMAIL_AUTH_CODE` | QQ 邮箱授权码 | 可选 |
| `EMAIL_DOWNLOAD_DIR` | 邮件下载目录 | 空（直接到 01 待分类） |

> **首次使用必填**：邮箱配置（如需邮件下载功能）

## 版本管理说明

### 版本号格式

本项目采用 **V1.0, V1.1, V2.0** 等版本号格式。

- **V1.0** = 初始发布（当前版本）
- **V1.1** = 小功能更新或 bug 修复
- **V2.0** = 重大功能更新

### 自动更新机制

每次运行任意脚本时，系统会自动检测是否有新版本：

1. 读取当前版本（`config.py` 中的 `SKILL_VERSION`）
2. 对比最新版本（`scripts/invoice-trip-organizer/VERSION`）
3. 若有新版本，自动备份旧脚本到 `.backup/<version>/`
4. 更新脚本文件，合并新增配置项
5. **绝不触碰用户数据**（发票、行程、台账等）

### 手动检查更新

```bash
python3 version_manager.py
```

### 版本回退

如果更新后出现问题，可从备份目录恢复：

```bash
cd scripts/invoice-trip-organizer/.backup/V1.0
cp *.py ../../../
```

## 常见问题

**Q: 不使用 Obsidian 可以用吗？**  
A: 可以。`OBSIDIAN_VAULT` 设为任意文件夹即可，目录结构会自动创建。

**Q: 邮箱下载不工作？**  
A: 确保使用授权码而非登录密码。163/126 → 设置 → POP3/SMTP/IMAP；QQ → 设置 → 账户 → POP3/IMAP/SMTP。

**Q: 发票金额/类别识别错误？**  
A: 把文件移入 `02 待核实/`，手动重命名为标准格式，下次运行会自动归档。

**Q: 如何多人使用？**  
A: 每位用户有独立的数据目录和 `config.py`，脚本共享同一个 Skill 目录。版本更新只更新脚本代码，不影响各用户的发票、行程、台账数据。
