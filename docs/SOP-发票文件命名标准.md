# SOP-发票文件命名标准

## 适用范围

本标准用于 `invoice-trip-organizer` 对个人发票文件进行识别、重命名、归档和行程关联。

## 标准命名格式

```text
YYYY-MM-DD_类别_金额[_位置][_发票号后4位]_MMDD_WB/YB_NNN.ext
```

字段说明：

| 字段 | 必填 | 说明 | 示例 |
|---|---:|---|---|
| `YYYY-MM-DD` | 是 | 发票日期或消费日期 | `2026-01-05` |
| `类别` | 是 | 发票类别 | `餐饮`、`住宿`、`机票` |
| `金额` | 是 | 保留两位小数 | `174.00` |
| `位置` | 否 | 交通类为路线，住宿类为酒店所在城市 | `广州-上海`、`中山` |
| `发票号后4位` | 否 | 用于区分同日同金额文件 | `1234` |
| `MMDD` | 否 | 操作日期 | `0105` |
| `WB/YB` | 否 | 报销状态，`WB` 未报，`YB` 已报 | `WB` |
| `NNN` | 否 | 三位流水号 | `031` |
| `ext` | 是 | 文件扩展名 | `pdf`、`ofd`、`jpg` |

## 类别口径

允许类别：

- 餐饮
- 住宿
- 机票
- 高铁
- 滴滴打车
- 行程单
- 高速费
- 充电费
- 礼品
- 结账单
- 其他
- 机票(保险)
- 滴滴打车(行程单)
- 住宿(结账单)
- 高速费(行程单)

## 示例

```text
2026-01-05_餐饮_174.00_0105_WB_031.pdf
2026-01-10_机票_1280.00_广州-上海_0110_WB_042.pdf
2026-01-12_住宿_264.00_中山_0112_WB_043.pdf
2026-03-24_住宿(结账单)_278.00_0324_WB_075.pdf
```

## 处理规则

1. 能识别日期、类别、金额的文件，自动归档到 `03 已完成/`。
2. 无法识别或命名不符合标准的文件，移动到 `02 待核实/`。
3. 用户在 `02 待核实/` 中按标准手动重命名后，脚本下次运行自动归档。
4. 同名文件不得覆盖，脚本应自动追加流水号。
5. `config.py`、邮箱授权信息、真实发票文件不进入 Git。
